Развернуть ваш сайт на GitHub Pages можно всего за несколько минут — это бесплатный хостинг для статических сайтов. Ваш проект (HTML, CSS, JS) идеально для этого подходит.



Вот пошаговая инструкция.



\### Вариант 1: Простой и быстрый (без командной строки)



Этот способ подойдёт, если вы не хотите разбираться с Git и консолью.



1\.  \*\*Создайте репозиторий:\*\*

&#x20;   \*   Зайдите на \[GitHub](https://github.com) и авторизуйтесь.

&#x20;   \*   Нажмите на зелёную кнопку \*\*"New"\*\* или на значок "+" в правом верхнем углу.

&#x20;   \*   В поле "Repository name" введите `username.github.io`, где `username` — это ваше имя пользователя на GitHub. Например, если ваш логин `ivanov`, то имя репозитория должно быть `ivanov.github.io`.

&#x20;   \*   Убедитесь, что репозиторий \*\*Public\*\* (публичный).

&#x20;   \*   Для простоты можно отметить галочку \*\*"Add a README file"\*\*. Нажмите \*\*"Create repository"\*\*.



2\.  \*\*Загрузите файлы сайта:\*\*

&#x20;   \*   Внутри созданного репозитория нажмите кнопку \*\*"Add file"\*\* → \*\*"Upload files"\*\*.

&#x20;   \*   Перетащите все файлы вашего проекта (`.html`, `.css`, `.js`, папки с картинками) в окно загрузки. \*\*Главная страница обязательно должна называться `index.html`\*\*.

&#x20;   \*   Внизу страницы добавьте сообщение коммита, например "Добавлены файлы сайта", и нажмите \*\*"Commit changes"\*\*.



3\.  \*\*Активируйте GitHub Pages:\*\*

&#x20;   \*   В вашем репозитории перейдите во вкладку \*\*"Settings"\*\* (Настройки).

&#x20;   \*   В левом меню найдите раздел \*\*"Pages"\*\*.

&#x20;   \*   В разделе "Build and deployment" (Сборка и развертывание) в выпадающем списке "Source" (Источник) выберите \*\*"Deploy from a branch"\*\* (Развертывать из ветки).

&#x20;   \*   Ниже, в поле "Branch" (Ветка), выберите `main` и нажмите \*\*"Save"\*\*.



Готово! Через 1-2 минуты ваш сайт будет доступен по адресу `https://username.github.io`.



\### Вариант 2: Продвинутый (с использованием Git)



Этот метод предпочтительнее для профессионалов, так как даёт больше контроля.



1\.  \*\*Клонируйте репозиторий\*\* на свой компьютер (команда `git clone`).

2\.  \*\*Скопируйте все файлы\*\* вашего сайта в эту локальную папку.

3\.  \*\*Отправьте изменения\*\* на GitHub:

&#x20;   ```bash

&#x20;   git add .

&#x20;   git commit -m "Первый коммит"

&#x20;   git push origin main

&#x20;   ```

4\.  Далее \*\*активируйте GitHub Pages\*\* через настройки репозитория, как описано в шагах 3.1-3.4 выше.



\### Альтернатива: Автоматическое развертывание с GitHub Actions



Этот способ удобен для более сложных проектов, но для вашего статического сайта он необязателен.



В настройках Pages (там же, где вы выбирали "Deploy from a branch") можно выбрать \*\*"GitHub Actions"\*\* в качестве источника. GitHub предложит готовые сценарии для сборки и публикации вашего сайта. Однако для простого HTML/CSS/JS сайта проще и быстрее использовать первый вариант.



\---



\*\*💡 Важные моменты:\*\*



\*   \*\*Главная страница:\*\* Убедитесь, что ваш файл `index.html` лежит в корне репозитория.

\*   \*\*Время:\*\* После первого опубликования сайт может быть доступен не сразу, подождите 1-2 минуты.

\*   \*\*Обновления:\*\* Чтобы обновить сайт после изменений, просто загрузите новые файлы в репозиторий (через веб-интерфейс или Git). GitHub Pages автоматически пересоберёт сайт.
